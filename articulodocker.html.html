Docker para principiantes: ¿Por qué usar contenedores?

Publicado el 2 de Noviembre, 2025 | Categoría: DevOps

"En mi máquina sí funciona". Esa es la frase que Docker vino a destruir. Contenerizar tus aplicaciones desde el día 1 te ahorrará horas de dolores de cabeza en los despliegues.

¿Qué es exactamente un contenedor?

Imagina un contenedor como una caja aislada que empaqueta tu aplicación y todo lo que necesita para ejecutarse (versión exacta de Python, librerías, sistema operativo base). Si esa caja se ejecuta bien en tu computadora portátil, funcionará exactamente igual en el servidor de la nube.

Ejemplo: Un Dockerfile simple para Python

Aquí tienes un archivo de configuración básico (llamado Dockerfile) para preparar y levantar una API:

# 1. Usar una imagen oficial de Python ligera
FROM python:3.10-slim

# 2. Establecer el directorio de trabajo dentro del contenedor
WORKDIR /app

# 3. Copiar las dependencias e instalarlas
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 4. Copiar todo el código fuente de tu aplicación
COPY . .

# 5. Comando para arrancar el servidor web
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]

El siguiente paso: Docker Compose

Una vez que domines los contenedores individuales, el siguiente nivel en la cultura DevOps es usar docker-compose.yml. Esto te permite levantar tu API, tu base de datos (como PostgreSQL) y tu sistema de caché (como Redis) al mismo tiempo con un solo comando: docker-compose up -d.
